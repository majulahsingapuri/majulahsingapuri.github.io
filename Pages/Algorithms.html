String Matching

Knot the easiest thing to do.

The Problem

String searching has been a problem that has been around years and there are many ways to go about looking for Substrings in a String. The problem given to us was to find all occurances of a Substring in a String of DNA. This is relatively easier as there are only 4 letters that can occur in a String of DNA. However, the complexity comes in the form of length. DNA chains are extremely long and a naive search can take extremely long to search through the String.

Approach

A quick look at Wikipedia reveals a treasure trove of already existing algorithms for use. We had to come up with 2 algorithms in a language of our choice. The first being a Brute Force algorithm and the second being an algorithm of our choice, be it a pre-existing one or one of our own creation. For the latter, we decided to implement the Knuth Morris Pratt algorithm as well as the Two-Way String Matching algorithm and see which one is the better one.

Implementation

The Brute Force was pretty straightforward to implement: have a for loop that runs down the length of the String and a while loop that checks through the search key until a mismatch, if any. It then prints out the locations of the occurances.

For the other 2 algorithms, the Two-Way String Matching turned out to be the trickier one as it requires a critical factorisation. It was also the only algorithm that didn't have a more modern implementation available on the web. I only found one version of it written in C, which meant that I had to decipher what it was trying to do and rewrite it in Python. Nicholas worked on the KMP algorithm implementation and it worked just as expected.

Analysis

Since this is an Algorithms module, the analysis of the algorithm, in terms of the preprocessing and actual run complexity, is just as important as the code. This is where Jethro and Timothy came in and did up a superb analysis of the algorithms. We decided to discard the Two-Way algorithm as the analysis was pretty complex and we had more uncertainties about analysing it as compared to the KMP. The effort wasn't gone to waste though as we all learnt something new from that venture.

Conclusion

This was a pretty interesting project as I got to see first hand what are the different methods of searching through Strings to find Substrings. As an end user you dont really need to think about such things when you're in an application and you press ⌘/Ctrl-F. As an introduction to basic Algorithms, this project was pretty informative as to the real world use of Algorithms.
